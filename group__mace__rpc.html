Detailed Description

The MACE.RPC Library leverages the features of Reflect Library, Stub Library, and CMT (Cooperative Multi-Tasking) Library to provide the most effecient, and native-feeling C++ RPC library known to man.

This library has three primary goals:

  • seamless interoperability with established RPC protocols like JSON-RPC or XML-RPC
  • seamless remote-object interface that fully abstracts the location of your C++ objects.
    • send a boost::function<> as an RPC parameter / return value and have them work transparently.
    • send cmt::stub::ptr<> as RPC parameters or return values.
    • propagate exceptions back across RPC boundary.
    • pass reference parameters.
    • use boost::signals<> over the network
  • abstract transport (udp,tcp,http,https,ssl) and protocol (json,binary,xml,etc) from the RPC interface.

Clearly if you plan on interacting with a well-defined JSON-RPC or JSON-XML interface then you will not be able to use the advanced features.

This library currently implements the JSON RPC-2.0 spec, but will expand to include XML-RPC, and various other protocols.

Dependencies

  • CMT (Cooperative Multi-Tasking) Library is used to handle multi-tasking and asynchronous operations.
  • Reflect Library is used for serailizing types.
  • Stub Library is used to provide native calling syntax.

Boost.ASIO is used for network communication.
